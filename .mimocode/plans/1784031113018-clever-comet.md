# Fix: Pipeline Crash — "Clip editing failed"

## Root Cause Analysis

The pipeline crash on July 13, 2026 (`daily_20260713_165510`) was caused by **multiple interacting bugs** that collectively caused `create_clip()` to return `None` after exhausting all 3 retry attempts.

### Primary Bugs (already fixed in commits after the failure)

| # | Bug | Commit | Status |
|---|-----|--------|--------|
| 1 | **Speed ramp produces clips shorter than 15s** — ffmpeg frame-boundary truncation at 30fps makes `-t 15.0` produce 14.967s, failing `CLIP_MIN_DURATION` validation | `3e41361` (+0.5s buffer) + `76e351c` (extend post segment) | FIXED |
| 2 | **Breath cut fade positioned beyond video end** — `apply_breath_cut` was adding `TEMP_REACTION_DURATION` (2s) on top of caller-provided duration, so `fade_start` exceeded the video length | `e64e0d8` | FIXED |
| 3 | **Color grade crashes on ffmpeg 8.1** — `convolution`, `noise`, `vignette` filters cause heap corruption (rc=2880417800) on ffmpeg 8.1 | `e64e0d8` + `328549a` (safe single-pass) | FIXED |
| 4 | **Workflow uses `cmd` shell** — "Terminate batch job (Y/N)?" prompt hangs/cancels the runner when ffmpeg subprocesses are interrupted | `e64e0d8` (switched to powershell) | FIXED |

### Remaining Issues (still present in current code)

These are hardening improvements that prevent future regressions:

#### A. `crop_to_shorts` has no retry logic
- **File**: `modules/clip_editor.py:504-609`
- **Problem**: If the ffmpeg call fails (corrupt source, unusual codec, OOM), the entire attempt fails immediately. All other steps (color, speed ramp, subtitles, audio, breath cut) have 2 retries + fallback, but crop has none.
- **Fix**: Add retry logic with a simple copy fallback.

#### B. Validation errors don't log actual values
- **File**: `modules/clip_validator.py:81-99`
- **Problem**: When validation fails, it says `duration 14s < 15s` but doesn't log the actual resolution or aspect ratio values. In `clip_editor.py:947`, the log message joins issues but doesn't include the actual file path or ffmpeg output.
- **Fix**: Add the file path and duration/dimensions to the validation log in `create_clip`.

#### C. No disk space check before pipeline start
- **File**: `run_pipeline.py`
- **Problem**: The pipeline can run out of disk space mid-edit. FFmpeg writes fail silently (output <10KB), causing every step to fail through its fallback chain, ultimately returning `None`.
- **Fix**: Check available disk space at pipeline start and fail early with a clear message.

#### D. Speed ramp `post_duration` can be 0
- **File**: `modules/clip_editor.py:84-85`
- **Problem**: `post_duration = min(max(TEMP_REACTION_DURATION, min(needed_post, available)), available)` — if `available` is 0 (source video too short), `post_duration` becomes 0, producing a clip that's just the ramp segments (very short).
- **Fix**: Add a minimum guard on `post_duration` and skip speed ramp entirely if the source is too short.

#### E. `_clean_work_dir` doesn't clean on crash
- **File**: `modules/clip_editor.py`
- **Problem**: If the pipeline crashes mid-attempt (e.g., ffmpeg timeout, OOM kill), leftover `_pipeline_*` directories accumulate in `assets/clips/`.
- **Fix**: Add cleanup of stale work directories at pipeline start.

## Implementation Plan

### Step 1: Add retry logic to `crop_to_shorts` (clip_editor.py)

**File**: `modules/clip_editor.py` — around line 504-609

Wrap the existing ffmpeg call in a retry loop (2 attempts). On second failure, try a simplified crop without the complex filter_complex (just `-ss`/`-t` seek + `-vf crop` + scale).

### Step 2: Improve validation logging (clip_editor.py + clip_validator.py)

**File**: `modules/clip_editor.py` — line 947

Change:
```python
print(f"  [editor] Validation failed: {'; '.join(last_issues)}", flush=True)
```
To:
```python
print(f"  [editor] Validation failed for {final_path}: {'; '.join(last_issues)}", flush=True)
dur = get_video_duration(final_path)
w, h = get_video_dimensions(final_path)
print(f"  [editor] Actual: {w}x{h}, {dur:.1f}s | Expected: {SHORTS_WIDTH}x{SHORTS_HEIGHT}, {CLIP_MIN_DURATION}-{CLIP_MAX_DURATION}s", flush=True)
```

### Step 3: Add disk space check (run_pipeline.py)

**File**: `run_pipeline.py` — after line 57 (before the pipeline starts)

Add a check for minimum 2GB free disk space. Fail early with a clear error message.

### Step 4: Guard speed ramp for short sources (clip_editor.py)

**File**: `modules/clip_editor.py` — line 84-85

Add a guard: if `available < 2.0` (less than 2s of post-ramp content available), skip the speed ramp entirely and just pass through the clip.

### Step 5: Clean stale work directories (clip_editor.py)

**File**: `modules/clip_editor.py` — inside `create_clip()`, before the retry loop (around line 710)

Add cleanup of any `_pipeline_*` directories older than 1 hour.
